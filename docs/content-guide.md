# コンテンツガイドライン

## 掲載基準

以下の条件を満たすものを語録として掲載する。

- 涼花みなせさん本人が配信・動画で発言したフレーズであること
- 複数の配信で使われているか、特に印象的な場面で使われたものであること
- 配信外（プライベートな発言など）のものは対象外

判断に迷う場合はモデレーター間で相談する。

## ファイルの作成方法

`content/quotes/` フォルダにMarkdownファイルを作成する。

**ファイル名のルール:**
- ローマ字表記を使う（例: `mushikera.md`、`otsumina.md`）
- スペースは使わず、単語をつなげるかハイフンで区切る
- すでに同じ語録のファイルがないか確認してから作成する

## Markdownのテンプレート

```markdown
---
title: 語録（日本語）
reading: よみがな
meaning: 一言で意味を説明
tags: [タグ1, タグ2]
rarity: 3
origin: 調査中
updated_at: "YYYY-MM-DD"
---

## 解説

語録が生まれた背景・経緯・エピソードを書く。

## 使用例

> セリフ<br>——YYYY年M月D日 涼花みなせ

## 関連リンク

- [配信タイトル](https://www.youtube.com/...)

情報提供者：ハンドルネーム
```

`origin` に配信リンクがある場合は以下の形式で記載する：

```
origin: "[配信タイトル](https://www.youtube.com/...)（YYYY年M月D日）"
```

## 各フィールドの説明

| フィールド | 必須 | 説明 |
|---|---|---|
| `title` | ○ | 語録そのもの |
| `reading` | ○ | ひらがなで読み方 |
| `meaning` | ○ | 短く分かりやすく意味を説明 |
| `tags` | ○ | 後述のタグ一覧から選ぶ |
| `rarity` | ○ | 1〜5の数字（後述） |
| `origin` | △ | 由来の配信タイトル・URL・日付を `[タイトル](URL)（日付）` の形式で記載。不明な場合は `調査中` |
| `updated_at` | △ | 掲載日・更新日。`YYYY-MM-DD` 形式。MDファイル作成時に今日の日付を入れる。大幅更新時も書き換える |

`## 使用例`・`## 関連リンク`・`情報提供者：` はMarkdown本文に自由に記載できる。省略可。

## 解説の書き方

語録の説明文は、**辞典らしい文体**を基本とします。

ちょっとした発言や些細なひと言であっても、もっともらしく・荘厳に説明してください。「なんでそんなに真面目に解説してるの？」というギャップが、くすっと笑えるエントリーを生みます。

**例（悪い）:**
> 冗談でも傷つく人がいるから気を付けようという意味。

**例（良い）:**
> 冗談のつもりでも2%の人は本気に受け取って傷ついてしまうと気づき、相手を責めるのではなく自分を省みた思慮深い言葉。

## レア度（rarity）の基準

| 値 | 目安 |
|---|---|
| 1 ★☆☆☆☆ | 新規ファン必見 |
| 2 ★★☆☆☆ | よく使う |
| 3 ★★★☆☆ | 印象的・記憶に残る |
| 4 ★★★★☆ | 知る人ぞ知る |
| 5 ★★★★★ | 激レア・名言・格言 |

主観でよい。迷ったら2か3に設定しておき、後から変更する。

## タグの一覧

自由に付けてよい。以下は現在使われているタグの例。

| タグ | 説明 |
|---|---|
| 挨拶 | 配信の開始・終了時の挨拶 |
| 感嘆詞 | 驚きや喜びを表す言葉 |
| 罵倒 | （愛情を込めた）悪口・ツッコミ |
| ゲーム | ゲーム配信中に生まれたフレーズ |
| コラボ | 他の配信者とのコラボで生まれたフレーズ |
| オリジナル曲 | オリジナル楽曲に関連するフレーズ |
| 生き様 | みなせさんの考え方・人生観を表す言葉 |
